# Модель доступа к данным

Сущность - список пожаланий

Роли пользователей относительно сущности:
- Автор
- Гость
- Пользователь

## Автор
Имеет доступ:
- просмотр списка
- редактирование элементов списка
- удаление элементов списка

## Гость
Имеет доступ:
- просмотр списка
- бронирование элементов списка
- освобождение ранее забронированных элементов списка

Роль Гость относительно списка пожалений пользователь становится по приглашению Автора

Гость может просматривать список сущностей, в которых он является Гостем

Автор может отозвать приглашение, после чего Гость становится обычным Пользователем и теряет доступ к просмотру списка пожеланий Автора

В случае, если в списке пожеланий Автора есть забронированные Гостем элементы списка - все бронирования освобождаются


## Пользователь
Все остальные пользователи не имеют доступа к списку пожеланий и не имеют доступа к информации о существовании чужих списков пожеланий
